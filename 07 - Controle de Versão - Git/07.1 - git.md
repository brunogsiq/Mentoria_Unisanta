# 07 - Controle de Versão (Git)

Conceitos básicos e fluxo comum para colaborar com Git.

Comandos essenciais
- `git clone <repo>`: clona repositório.
- `git checkout -b feature/x`: cria e troca para nova branch.
- `git add` / `git commit -m "msg"`: salvar mudanças localmente.
- `git push origin branch`: enviar para remoto.
- Pull Request / Merge: revisão e integração de código.

Estratégias
- Git Flow: ramificação com `develop`/`release`/`hotfix`.
- Trunk Based: pequenas branches curtas que integram direto no `main`.

Exemplo prático
1. `git checkout -b feature/login`
2. Implementar a feature, adicionar testes.
3. `git add . && git commit -m "login: add auth"`
4. `git push origin feature/login` e abrir PR.

Dica
- Faça commits atômicos e mensagens claras; isso facilita revisão e revert.
