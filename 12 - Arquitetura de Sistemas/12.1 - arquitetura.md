# 12 - Arquitetura de Sistemas

Visão inicial sobre padrões de arquitetura e camadas.

Modelos comuns
- Monolito: aplicação única, fácil de começar.
- Microsserviços: serviços pequenos e independentes, melhor escalabilidade.

Camadas típicas
- Controller (entrada), Service (regras), Repository (acesso a dados).

Fluxo simples
Frontend → API Gateway → Serviço → Banco → Resposta

Critérios para escolher
- Se o produto é pequeno e com equipe reduzida: monolito.
- Se houver necessidade de escalar partes isoladas: microsserviços.

Dica
- Comece simples; só migre para arquiteturas mais complexas quando houver necessidade clara.
