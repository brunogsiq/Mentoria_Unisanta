# 02.2 - Papéis e Responsabilidades (Template)

Modelo de descrição de papéis para uso em squads:

- **Responsabilidade**: resumo do que faz.
- **Entradas**: artefatos necessários para execução.
- **Saídas**: entregáveis esperados.
- **Critérios de aceite**: como validar trabalho concluído.

Exemplo prático:
- **Backend**: entrega APIs estáveis; entradas: especificação da API; saídas: endpoints implementados e testados; critérios: cobertura unitária e integração.

Use este template para criar fichas de papéis para cada membro do time.
