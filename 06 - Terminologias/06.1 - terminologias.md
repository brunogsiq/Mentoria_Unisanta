# 06 - Terminologias

Glossário prático — definições rápidas que você verá com frequência:

- Bug: comportamento que difere do esperado.
- Issue / Task: unidade de trabalho.
- Feature: nova funcionalidade.
- Release / Deploy: entrega de versão para um ambiente.
- Build: processo que transforma código em artefato executável.
- Commit / Merge / PR: fluxo de colaboração com Git.
- Pipeline: sequência automatizada de build/test/deploy.
- Environment: Dev, QA, Staging, Prod — níveis de implantação.
- Hotfix: correção urgente em produção.

Exemplo: ao abrir um PR, descreva o objetivo, inclua testes e marque reviewers.
