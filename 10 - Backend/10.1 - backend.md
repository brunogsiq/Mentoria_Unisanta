# 10 - Backend

Visão prática do backend e responsabilidades comuns.

O que faz o backend
- Processa regras de negócio.
- Expõe APIs (REST/GraphQL) para o frontend.
- Gerencia persistência (banco) e autenticação.

Exemplo mínimo (endpoint REST)
- `POST /login` recebe credenciais, valida no banco e retorna token.

Escolha de tecnologias
- Node.js (rápido para I/O), Python (simplicidade), Java/C# (apps corporativos).

Boas práticas
- Valide dados de entrada, use logs e trate erros de forma consistente.
- Escreva testes unitários e de integração para endpoints críticos.
