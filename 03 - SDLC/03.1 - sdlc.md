# 03 - SDLC (Ciclo de Vida de Desenvolvimento)

O SDLC descreve as fases principais para construir e manter um software. Abaixo um resumo objetivo com exemplos práticos.

Fases principais
- Planejamento: definir escopo e estimativas.
- Análise de requisitos: mapear o que o produto deve fazer (regras de negócio, critérios de aceite).
- Desenvolvimento: implementação de funcionalidades.
- Testes: validação (unit, integration, e2e) para garantir qualidade.
- Deploy: entrega do software para um ambiente (staging/prod).
- Manutenção: correções, ajustes e novas funcionalidades.

Exemplo simples de fluxo
1. Equipe recebe uma feature: "Cadastro de usuário".
2. PO especifica critérios de aceite e o time estima (planning).
3. Dev implementa endpoint `POST /users` e a tela de cadastro.
4. QA executa testes manuais e automatizados; um bug é encontrado e corrigido.
5. Após revisão, a feature é deployada para produção.

STLC (Ciclo de Testes)
- Planejamento de testes: definir escopo dos testes para a release.
- Criação de cenários: escrever casos de teste com passos e dados.
- Execução: rodar os testes (manuais/automáticos).
- Reporte: registrar falhas e acompanhar correções.

Dicas práticas
- Integre testes ao pipeline (CI) para detectar regressões cedo.
- Faça deploy contínuo em pequenos passos quando possível (release frequent).
- Mantenha documentação mínima com exemplos de uso das APIs para facilitar testes.
