# 13 - APIs e Comunicação

Conceitos essenciais para projetar e usar APIs.

REST e CRUD
- Operações básicas: Create (POST), Read (GET), Update (PUT/PATCH), Delete (DELETE).
- Use JSON como formato padrão para troca de dados.

Status HTTP (exemplos)
- 200 OK — sucesso
- 201 Created — recurso criado
- 400 Bad Request — validação falhou
- 401 Unauthorized — sem autenticação
- 404 Not Found — recurso não existe

Ferramentas úteis
- Swagger/OpenAPI: documentar API.
- Postman/Insomnia: testar endpoints manualmente.

Exemplo de endpoint
- `GET /products` → retorna lista de produtos JSON.

Dica
- Documente os contratos (OpenAPI) para que frontend e backend usem a mesma referência.
