# 01 - Introdução à Tecnologia

Esta seção oferece um panorama prático e direto sobre conceitos fundamentais de tecnologia, com exemplos simples para orientar estudos e aplicações.

1) Visão geral: como um sistema funciona
- Frontend: interface com o usuário (site, app).
- Backend: servidor que processa regras e dados.
- Banco de dados: guarda informações (usuários, pedidos).

Exemplo simples (fluxo):
1. Usuário clica em "Entrar" no app (frontend).
2. O frontend envia uma requisição ao backend: `POST /login` com email/senha.
3. Backend valida as credenciais no banco de dados.
4. Se válido, backend retorna um token ao frontend; o usuário entra no app.

2) Conceitos rápidos
- API: ponto de comunicação entre frontend e backend. Ex.: `GET /products` retorna lista de produtos.
- CRUD: Create, Read, Update, Delete — operações básicas sobre dados.

Exemplo CRUD (pseudo):
- `POST /items` → cria um item
- `GET /items/1` → lê o item com id 1
- `PUT /items/1` → atualiza o item 1
- `DELETE /items/1` → remove o item 1

3) Testes (por que são importantes)
- Testes manuais: validar funcionalidades passo a passo.
- Testes automatizados: scripts que conferem comportamento (unit, integration, e2e).

Exemplo prático: teste unitário de função que soma dois números — garante que a lógica continua correta após mudanças.

4) Metodologias de trabalho (breve)
- Scrum e Kanban: frameworks para organizar entregas por iterações e fluxo de trabalho.

5) Ferramentas comuns
- Controle de versão: `git` (clone, commit, push).
- Gerenciamento de tarefas: Jira, Azure DevOps.
- IDEs: VS Code, IntelliJ.

6) Próximos passos sugeridos
- Experimente criar um pequeno backend (por exemplo, em Node.js ou Python) que exponha uma API CRUD simples.
- Crie um repositório Git, faça commits pequenos e abra um README com instruções de uso.

Se quiser, eu posso gerar um exemplo mínimo de API (Node.js/Express ou Python/Flask) e um guia passo-a-passo para você rodar localmente.
