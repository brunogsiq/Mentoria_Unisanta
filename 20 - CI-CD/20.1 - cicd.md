# 20 - CI/CD

Princípios para integrar e entregar software de forma contínua.

Fluxo típico
1. Commit → 2. Build → 3. Testes → 4. Deploy (staging/prod)

Ferramentas
- GitHub Actions, GitLab CI, Jenkins, Azure Pipelines.

Exemplo básico (GitHub Actions)
- Workflow que roda testes e, se passarem, gera build e publica imagem/container.

Dica
- Separe pipelines por propósito (PR checks vs Release pipeline) e mantenha passos curtos e rápidos.
