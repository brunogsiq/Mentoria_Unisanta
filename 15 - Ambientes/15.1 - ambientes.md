# 15 - Ambientes

Descrição objetiva dos ambientes de software e práticas de deploy.

Ambientes típicos
- Dev: ambiente de desenvolvimento (integração local).
- QA/Staging: ambiente para testes integrados.
- Prod: ambiente de produção, acessado por usuários.

Práticas de deploy
- Automatize deploys com pipeline (CI/CD).
- Mantenha configurações por ambiente (variáveis de ambiente, secrets).

Exemplo simples
- Use `feature branches` para desenvolvimento; integre em `main` via PR e deixe pipeline automatizar build/test/deploy para staging.
